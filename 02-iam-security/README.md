# 02 - IAM Security

## Objetivo

Estudar os fundamentos de segurança de identidade e acesso na AWS usando IAM, com foco em usuários, grupos, roles, policies, MFA e princípio do menor privilégio.

## Conteúdos estudados

- AWS IAM
- IAM Users
- IAM Groups
- IAM Roles
- IAM Policies
- MFA
- Access Keys
- Principle of Least Privilege
- Permission Boundaries
- Segurança de credenciais

## Conceitos principais

IAM é o serviço da AWS usado para controlar quem pode acessar quais recursos e com quais permissões.

O objetivo deste módulo é entender como evitar permissões excessivas, reduzir riscos de credenciais expostas e aplicar boas práticas de controle de acesso.

## Laboratórios planejados

- Criar um usuário IAM de laboratório
- Criar um grupo com permissões limitadas
- Criar uma policy personalizada
- Ativar MFA
- Comparar permissões de usuário, grupo e role
- Simular o risco de uma policy muito permissiva
- Documentar o princípio do menor privilégio
